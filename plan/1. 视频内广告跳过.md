# B站浏览器端广告自动跳过技术方案

## 1️⃣ 项目目标

* 自动识别 B站视频中植入广告（如拼多多、转转、妙界等）
* 自动跳过广告段，无需用户手动输入或标记
* 仅供个人使用，轻量化、性能可控

---

## 2️⃣ 核心思路

广告识别依赖**三重信号融合**：

| 信号来源     | 描述                         | 技术实现                           |
| -------- | -------------------------- | ------------------------------ |
| 弹幕关键词    | 广告弹幕频繁出现关键词，如“广告”“恰饭”“拼多多” | 监听弹幕 DOM，统计关键词出现次数             |
| 字幕/语音关键词 | 视频中 UP 主口播广告，如“本期视频由拼多多赞助” | 使用 Web Speech API 或抓取 B站自带字幕节点 |
| 广告长度经验   | 大部分广告固定长度 15~60 秒          | 检测到广告开头后，跳过一定时间或监听结束信号         |

**判定逻辑**：

* 当 **两个信号触发** → 判定广告开始
* 到达广告结束时间或结束信号 → 跳过至广告结束

## 3️⃣ 技术方案架构

### 3.1 插件/脚本结构

```text
bilibili-ad-skipper/
├─ manifest.json        # 浏览器插件配置
├─ content.js           # 注入脚本，核心逻辑
├─ utils.js             # 工具函数（跳转时间、日志、配置）
└─ config.json          # 用户可配置参数（关键词列表、跳过时长）
```

### 3.2 核心模块设计

1. **视频监控模块**

   * 获取 `<video>` 标签
   * 定时或事件触发读取 `currentTime`

2. **广告识别模块**

   * 弹幕关键词识别（DOM / MutationObserver）
   * 字幕/语音关键词识别（Web Speech API 或字幕节点）
   * 经验时长规则（检测广告长度后跳转）

3. **跳过控制模块**

   * 一旦判定广告，调用：

   ```js
   video.currentTime = adEndTime;
   ```

   * 可监听 `seeked` 事件，确保平滑跳转

4. **配置模块**

   * 可自定义关键词列表
   * 可调整默认跳过时长（例如 30 秒）

## 4️⃣ 核心逻辑示例（伪代码）

```js
const video = document.querySelector("video");
let adDetected = false;
let adStartTime = 0;

function detectAdFromDanmaku() {
    const danmus = document.querySelectorAll(".bilibili-player-danmaku .bilibili-danmaku-item");
    let count = 0;
    danmus.forEach(d => {
        if (/广告|恰饭|拼多多|转转|妙界/.test(d.textContent)) count++;
    });
    return count > 3; // 弹幕频率阈值
}

function detectAdFromSubtitle() {
    const subtitles = document.querySelectorAll(".bilibili-player-subtitle");
    return Array.from(subtitles).some(s => /感谢拼多多|感谢转转|本期视频由/.test(s.textContent));
}

function skipAd() {
    if (adDetected) {
        video.currentTime = adStartTime + 30; // 默认广告长度 30 秒
        adDetected = false;
    }
}

// 主循环
setInterval(() => {
    if (detectAdFromDanmaku() || detectAdFromSubtitle()) {
        adDetected = true;
        adStartTime = video.currentTime;
    }
    skipAd();
}, 500);
```

## 5️⃣ 优化建议

1. **性能优化**

   * 弹幕/字幕用 `MutationObserver` 监听新增节点，避免遍历全量 DOM
   * 画面识别或 OCR 可延迟到广告频繁触发时再做

2. **误判减少**

   * 采用两信号以上触发才判定广告
   * 可增加短缓冲时间（例如检测到广告后延迟 0.5 秒再跳）

3. **广告结束检测**

   * 除默认时长，可监听字幕/语音出现“正式内容开始”或“广告结束”关键词
   * 避免跳过正常内容

4. **配置灵活**

   * 用户可添加自定义广告关键词
   * 调整跳过默认时间、检测频率

## 6️⃣ 开发路线

1. **第一阶段 MVP**

   * 弹幕 + 字幕关键词识别
   * 默认跳过固定时长广告

2. **第二阶段**

   * 集成 Web Speech API 进行语音实时识别
   * 可根据广告结束关键词智能跳过

3. **第三阶段（可选）**

   * 图像识别广告 logo，提高静音广告检测能力
   * 弹幕/语音/画面三信号融合，精度更高

---

## 7️⃣ 风险与注意事项

* 仅供个人使用，避免发布或共享广告跳过数据
* B站广告频率和植入形式不固定，算法需灵活可调整
* 高性能实时识别需注意 CPU/内存占用
